# 创建新短剧项目

初始化一个AI短剧项目，创建项目配置文件和标准目录结构。

## 用法

```bash
/drama-new [剧名]
/drama-new [剧名] --episodes [集数]
/drama-new [剧名] --episodes [集数] --type [类型]
```

## 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 剧名 | 短剧标题 | "时空快递员" |
| --episodes | 集数（默认5集） | --episodes 10 |
| --type | 类型标签 | --type 都市言情 |

## 执行流程

### Step 1: 确认项目位置

询问用户项目存放位置：
- 当前目录
- 新建子目录（以剧名命名）

### Step 2: 创建目录结构

```
/项目目录/
├── 00_项目配置.md              # 项目元信息和进度追踪
├── 01_故事大纲.md              # 三幕结构 + 分集大纲
│
├── 02_角色设定/                # 角色设定卡
│   └── .gitkeep
│
├── 03_角色图片/                # 角色成品图 ★ 一致性锚点
│   └── .gitkeep
│
├── 04_场景设定/                # 场景设定卡
│   └── .gitkeep
│
├── 05_场景图片/                # 场景成品图 ★ 一致性锚点
│   └── .gitkeep
│
├── 06_分镜脚本/                # 分镜表格
│   └── .gitkeep
│
├── 07_视频提示词/              # 视频提示词（含参考图）
│   └── .gitkeep
│
├── 08_配音台本/                # 传统模式专用
│   └── .gitkeep
│
└── 99_导出包/                  # 最终输出
    └── .gitkeep
```

询问用户基本创作需求：

- 目标受众
- 风格参考

## 示例

```bash

✅ 项目配置已生成

📋 下一步：
1. 使用 /drama-plan 生成故事大纲
2. 使用 /drama-character 创建角色设定
3. 使用 /drama-scene 创建场景设定
```

## 下一步

项目创建后，按以下顺序执行：

```bash
/drama-plan              # 生成故事大纲
/drama-character --all   # 创建所有角色设定
/drama-scene --all       # 创建所有场景设定
```
