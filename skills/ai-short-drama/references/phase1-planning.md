# 阶段一：策划工作流

## 目标

完成短剧的整体策划，产出可执行的故事框架。

## 输入

- 用户的初始创意或需求

## 输出

- `01_故事大纲.md`
- `00_项目配置.md`

---

## 执行步骤

### Step 1: 收集需求

向用户确认以下信息：

```
请提供以下信息（可部分提供，我会帮你完善）：

1. 【剧名/暂定名】
2. 【类型】（都市/古装/科幻/悬疑/爱情/喜剧/其他）
3. 【集数】和【单集时长】
4. 【一句话故事概念】
5. 【目标受众】
6. 【参考作品】（如有）
7. 【核心情绪】（想给观众什么感受）
8. 【特殊要求】（如有商业植入、平台要求等）
```

### Step 2: 生成故事大纲

基于用户输入，生成完整的 `01_故事大纲.md`：

1. **核心概念提炼**
   - 确认一句话故事
   - 明确核心冲突
   - 提炼情感内核

2. **世界观构建**
   - 时间背景
   - 空间背景
   - 特殊设定（如有）

3. **角色框架**
   - 主要角色列表
   - 各角色的核心欲望和阻碍
   - 角色关系网络

4. **剧情弧线规划**
   - 第一幕：铺垫（比例约20%）
   - 第二幕：冲突（比例约60%）
   - 第三幕：解决（比例约20%）

5. **分集大纲**
   - 每集的标题、核心事件、情绪基调、结尾钩子

### Step 3: 确定视觉风格

与用户确认：

```
基于故事内容，建议以下视觉风格：

【整体风格】：[建议]
【色调倾向】：[建议]
【参考作品】：[建议]

是否符合预期？是否需要调整？
```
