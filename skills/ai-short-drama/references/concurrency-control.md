# 并发控制策略

## 概述

本文档定义了 AI 短剧插件各阶段的并发控制策略，以避免 API 限流和提高处理效率。

---

## 并发限制配置

### 全局限制

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **MAX_CONCURRENT_REQUESTS** | 3 | 同时进行的最大请求数 |
| **BATCH_SIZE_LARGE** | 5 | 大型任务批次大小（分镜、提示词） |
| **BATCH_SIZE_MEDIUM** | 3 | 中型任务批次大小（角色设计） |
| **BATCH_SIZE_SMALL** | 10 | 小型任务批次大小（配音台本） |
| **REQUEST_INTERVAL_MS** | 1000 | 批次间间隔时间（毫秒） |

### 各阶段并发配置

| 阶段 | 并发数 | 批次大小 | 间隔时间 | 说明 |
|------|--------|----------|----------|------|
| 策划阶段 | 1 | - | - | 单次请求，无需并发控制 |
| 角色设计 | 1 | 3 角色 | 1000ms | 角色间需保持风格一致 |
| 分镜脚本 | 3 | 5 镜头 | 1000ms | 批量生成，控制并发 |
| 补充提示词 | 3 | 5 镜头 | 1000ms | 批量补充，避免限流 |
| 分镜审核 | 3 | 10 镜头 | 500ms | 主要是读取，可适当提高 |
| 配音台本 | 3 | 20 台词 | 1000ms | 台词独立性强，可大批量 |
| 导出阶段 | 1 | - | - | 单次请求，无需并发控制 |

## 批量处理策略

### 策略1：顺序处理（低并发场景）

**适用场景：**
- 内容复杂，需要精心设计
- 上下文关联性强
- 质量优先于速度

**执行方式：**
```
for item in items:
    result = process(item)  # 逐个处理
    save(result)
```

**使用阶段：**
- 策划阶段（故事大纲）
- 角色设计（角色间风格需一致）

---

### 策略2：批量顺序处理（推荐）

**适用场景：**
- 内容相对独立
- 需要控制 API 调用频率
- 平衡质量和效率

**执行方式：**
```
batches = split_into_batches(items, BATCH_SIZE)
for batch in batches:
    results = process_batch(batch)  # 一次处理多个
    for result in results:
        save(result)
    sleep(REQUEST_INTERVAL_MS)  # 批次间间隔
```

**使用阶段：**
- 分镜脚本生成
- 补充提示词
- 分镜审核

---

### 策略3：大批量处理（高效率场景）

**适用场景：**
- 任务独立性强
- 内容格式化程度高
- 效率优先

**执行方式：**
```
batches = split_into_batches(items, BATCH_SIZE_LARGE)
for batch in batches:
    prompt = build_batch_prompt(batch)  # 构建批量提示词
    results = llm_call(prompt)  # 一次调用返回多个结果
    save_all(results)
    sleep(REQUEST_INTERVAL_MS)
```

**使用阶段：**
- 配音台本生成

## 限流恢复策略

### 检测限流

当出现以下情况时，判定为限流：
- API 返回 429 状态码
- 响应时间异常长（> 30秒）
- 返回 "rate limit exceeded" 错误

### 恢复流程

1. **立即停止**：暂停当前批次处理
2. **指数退避**：等待时间按指数增长
   - 第一次：等待 2 秒
   - 第二次：等待 4 秒
   - 第三次：等待 8 秒
   - 最大等待：60 秒
3. **降低并发**：将并发数减半
4. **重试**：重新发送请求
5. **记录**：记录限流事件，用于调整配置

### 代码示例

```javascript
async function withRetry(fn, maxRetries = 3) {
  let delay = 2000
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn()
    } catch (error) {
      if (error.status === 429 && i < maxRetries - 1) {
        console.log(`限流，等待 ${delay}ms 后重试...`)
        await sleep(delay)
        delay = Math.min(delay * 2, 60000)
      } else {
        throw error
      }
    }
  }
}
```

## 配置调整建议

### 根据 API 限流情况调整

| 限流情况 | 调整建议 |
|----------|----------|
| 频繁限流 | 降低 BATCH_SIZE 至 2-3，增加 REQUEST_INTERVAL_MS 至 2000ms |
| 偶尔限流 | 维持默认配置，启用指数退避 |
| 无限流 | 可适当提高 BATCH_SIZE 至 8-10 |

### 根据任务规模调整

| 任务规模 | BATCH_SIZE | 并发数 |
|----------|------------|--------|
| 小型（<20个镜头） | 3-5 | 2 |
| 中型（20-50个镜头） | 5 | 3 |
| 大型（>50个镜头） | 5-8 | 3 |
