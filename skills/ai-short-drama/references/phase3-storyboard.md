# 阶段三：分镜脚本工作流

## 目标

将故事转化为具体的分镜脚本，并为每个镜头生成即梦AI视频提示词。

> 💡 **Seedance 2.0 融合模式**：支持在提示词中直接融合配音信息，一键生成带配音的视频！

## 输入

- `01_故事大纲.md`
- `02_角色设定/`

## 输出

- `03_分镜脚本/第X集.md` × N
- `04_视频提示词/第X集/` × N

---

## 执行步骤

### Step 1: 选择生成模式

与用户确认：

```
生成模式选择：

1. 【Seedance 2.0 融合模式】（推荐）
   - 提示词包含画面+配音+音频
   - 一键生成带配音视频
   - 无需后续对口型
   - 适合：有台词的剧情短剧

2. 【传统分步模式】
   - 仅生成画面提示词
   - 后续需要单独配音
   - 适合：无声画面/后期单独配音

请选择生成模式（1/2）：
```

### Step 2: 确定分镜策略

与用户确认：

```
分镜策略选择：

1. 【镜头密度】
   - 紧凑型：每分钟约8-10个镜头
   - 标准型：每分钟约5-7个镜头
   - 舒缓型：每分钟约3-5个镜头

2. 【叙事风格】
   - 对话驱动：多中近景，强调人物互动
   - 动作驱动：多动感镜头，强调运动感
   - 氛围驱动：多远景特写切换，强调情绪渲染

3. 【镜头时长偏好】
   - 快节奏：平均2-3秒/镜头
   - 中等：平均4-5秒/镜头
   - 慢节奏：平均6秒以上/镜头

请选择或说明您的偏好：
```

### Step 3: 逐集拆分分镜

对每一集执行：

#### 3.1 生成本集概要

```markdown
# 第X集：[标题]

## 集数信息
| 项目 | 内容 |
|------|------|
| 时长 | 秒 |
| 镜头数 | 个 |
| 核心事件 | |
| 情绪曲线 | |
| 生成模式 | Seedance 2.0 融合模式 / 传统分步模式 |

## 剧情梗概
[200字以内的本集剧情概述]
```

#### 3.2 拆解镜头清单

```markdown
## 镜头清单

| 编号 | 场景 | 角色 | 动作 | 台词 | 时长 | 情绪 |
|------|------|------|------|------|------|------|
| S01_C01 | | | | | | |
| S01_C02 | | | | | | |
| ... | | | | | | |
```

#### 3.3 详细分镜描述 + 提示词生成（关键步骤）

> ⚠️ **重要规则**：每个镜头**必须同时生成**分镜描述和提示词，不可跳过！

对每个镜头执行以下流程：

**Step A: 生成分镜描述**
```markdown
### S01_C01 [简短描述]

**场景：** [详细场景描述]
**时间：** [时间]
**角色：** [出场角色及状态]
**动作：** [具体动作描述]
**情绪：** [角色情绪 + 画面氛围]
**运镜：** [景别 + 运镜方式]
**时长：** 秒
```

**Step B: 立即生成提示词（不可跳过）**

**【Seedance 2.0 融合模式】**
```markdown
**Seedance 2.0 融合提示词：**
```
Subject: [主体描述]
Action: [动作描述]
Camera: [运镜描述]
Style: [风格描述]

Voice:
- 角色：[说话角色]
- 声音：[声音特点]
- 台词：「[台词内容]」
- 情绪：[情绪描述]
- 语速：[正常/偏快/偏慢]

Audio:
- 环境音：[环境声音]
- 配乐：[背景音乐风格]
- 音效：[特殊音效]

Constraints: [负面提示]，[时长]
```

**【传统分步模式】**
```markdown
**即梦AI视频提示词：**
```
[完整提示词，格式：场景 + 主体动作 + 风格 + 情绪 + 运镜]
```
```

**Step C: 自检（每个镜头完成后立即检查）**

**【融合模式检查项】**
```
□ 画面提示词是否已生成？
□ 配音信息是否完整（如需配音）？
□ 音频氛围是否已填写？
□ 提示词格式是否符合 Seedance 2.0 规范？
```

**【传统模式检查项】**
```
□ 提示词是否已生成？
□ 提示词是否包含场景描述？
□ 提示词是否包含角色/主体动作？
□ 提示词是否包含视觉风格？
□ 提示词是否包含运镜描述？
```

> 💡 **提示**：如果任何一个检查项为 ❌，必须立即补充，不要继续下一个镜头！

## 提示词质量检查

**【融合模式检查项】**
```
✅ 结构完整：包含 Subject, Action, Camera, Style
✅ 配音完整：Voice 字段信息齐全（如需配音）
✅ 音频合理：Audio 字段与场景氛围匹配
✅ 时长匹配：提示词内容量与镜头时长相符
✅ 连贯性：与前后镜头能自然衔接
✅ 口型同步：Constraints 包含口型同步要求
```

**【传统模式检查项】**
```
✅ 结构完整：包含场景、主体、动作、风格、情绪
✅ 描述具体：避免模糊词汇，使用具体描述
✅ 时长匹配：提示词内容量与镜头时长相符
✅ 连贯性：与前后镜头能自然衔接
✅ 可执行性：避免AI难以理解的动作
```

## 完成标准

- [ ] 所有集数都有完整的分镜脚本
- [ ] 每个镜头都有详细的分镜描述
- [ ] 每个镜头都有可用的视频提示词
- [ ] **【融合模式】** 配音信息已融合到提示词中
- [ ] 提示词已按集汇总，方便复制使用
- [ ] 用户已确认分镜内容

---

## ⚠️ 重要：进入审核环节

分镜脚本和提示词生成完成后，**必须进入审核环节**确保一致性。

进入 `phase3.5-review.md` 进行分镜与提示词的审核。

---

## 后续步骤

### 【融合模式】
审核通过后，**可直接进入导出阶段**：
```
/drama-storyboard → /drama-review → /drama-export
```
> 💡 融合模式已包含配音信息，无需单独生成配音台本

### 【传统模式】
审核通过后，需要生成配音台本：
```
/drama-storyboard → /drama-review → /drama-script → /drama-export
```

---

## 审核环节概览

审核环节将检查以下内容：

| 维度 | 检查项 |
|------|--------|
| 内容一致性 | 场景、角色、动作、时间、情绪 |
| 技术规范性 | 运镜匹配、时长合理、格式规范 |
| 连贯性 | 镜头衔接、角色状态、场景连续 |
| **【融合模式】配音一致性** | 台词与分镜匹配、情绪符合场景 |
