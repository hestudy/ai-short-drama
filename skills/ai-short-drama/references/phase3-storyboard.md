# 阶段三：分镜脚本工作流

## 目标

将故事转化为具体的分镜脚本，并为每个镜头生成即梦AI视频提示词。

> 💡 **Seedance 2.0 融合模式**：支持在提示词中直接融合配音信息，一键生成带配音的视频！

## 输入

- `01_故事大纲.md`
- `02_角色设定/`

## 输出

- `03_分镜脚本/第X集.md` × N
- `04_视频提示词/第X集/` × N

---

## ⚠️ 并发控制配置

> 🔒 **重要**：本阶段是调用频率最高的环节，必须严格遵守并发限制！

| 参数 | 值 | 说明 |
|------|-----|------|
| **批次大小** | 5 个镜头/批 | 每批处理的镜头数量 |
| **并发数** | 3 | 同时进行的最大请求数 |
| **间隔时间** | 1000ms | 批次之间的等待时间 |
| **建议模式** | 批量顺序处理 | 平衡质量和效率 |

**预估调用次数对比：**

| 方式 | 50个镜头调用次数 | 说明 |
|------|------------------|------|
| 逐个处理 | 100 次 | 每镜头2次调用（分镜+提示词） |
| 批量处理 | 10 次 | 每批5个，共10批 |
| **节省** | **90%** | 大幅降低限流风险 |

详细配置参见：[concurrency-control.md](./concurrency-control.md)

## 执行步骤

### Step 1: 选择生成模式

与用户确认：

```
生成模式选择：

1. 【Seedance 2.0 融合模式】（推荐）
   - 提示词包含画面+配音+音频
   - 一键生成带配音视频
   - 无需后续对口型
   - 适合：有台词的剧情短剧

2. 【传统分步模式】
   - 仅生成画面提示词
   - 后续需要单独配音
   - 适合：无声画面/后期单独配音

请选择生成模式（1/2）：
```

### Step 2: 确定分镜策略

与用户确认：

```
分镜策略选择：

1. 【镜头密度】
   - 紧凑型：每分钟约8-10个镜头
   - 标准型：每分钟约5-7个镜头
   - 舒缓型：每分钟约3-5个镜头

2. 【叙事风格】
   - 对话驱动：多中近景，强调人物互动
   - 动作驱动：多动感镜头，强调运动感
   - 氛围驱动：多远景特写切换，强调情绪渲染

3. 【镜头时长偏好】
   - 快节奏：平均2-3秒/镜头
   - 中等：平均4-5秒/镜头
   - 慢节奏：平均6秒以上/镜头

请选择或说明您的偏好：
```

## 提示词质量检查

**【融合模式检查项】**
```
✅ 结构完整：包含 Subject, Action, Camera, Style
```

**【传统模式检查项】**
```

✅ 描述具体：避免模糊词汇，使用具体描述
✅ 时长匹配：提示词内容量与镜头时长相符
✅ 连贯性：与前后镜头能自然衔接
✅ 可执行性：避免AI难以理解的动作
```

## 限流恢复策略

如果遇到 API 限流：

1. **暂停处理**：停止当前批次
2. **指数退避**：
   - 第一次等待 2 秒
   - 第二次等待 4 秒
   - 最大等待 60 秒
3. **调整配置**：
   - 将批次大小从 5 降至 3
   - 将间隔时间从 1000ms 增至 2000ms
4. **重试**：重新发送请求
