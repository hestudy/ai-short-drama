# 阶段二：角色设计工作流

## 目标

为所有主要角色创建完整的设定卡，包括可直接用于即梦AI的图片生成提示词。

## 输入

- `01_故事大纲.md`

## 输出

- `02_角色设定/[角色名].md` × N
- `02_角色设定/角色外观提示词.md`

---

## ⚠️ 并发控制配置

> 🔒 **重要**：本阶段需要为多个角色生成提示词，需遵守并发限制！

| 参数 | 值 | 说明 |
|------|-----|------|
| **处理模式** | 顺序处理 | 角色间需保持风格一致 |
| **角色间隔** | 1000ms | 每个角色完成后等待 |
| **表情提示词** | 批量生成 | 每个角色的多个表情一次生成 |

**设计原则：**
- 角色之间需要风格统一，建议逐个处理
- 每个角色的多个表情提示词可以一次生成
- 处理完一个角色后等待再处理下一个

详细配置参见：[concurrency-control.md](./concurrency-control.md)

---

```

- 角色D

【反派】
```

## 限流恢复策略

如果遇到 API 限流：

1. **暂停处理**：停止当前角色
2. **指数退避**：等待 2-60 秒
3. **增加间隔**：将角色间隔从 1000ms 增至 2000ms
4. **重试**：重新发送请求
