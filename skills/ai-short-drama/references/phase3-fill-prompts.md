# 阶段三·补：补充缺失提示词

## 目标

检查并补充分镜脚本中缺失或不完整的AI视频提示词。

## 触发命令

```
/ai-short-drama fill-prompts
```

## 输入

- `03_分镜脚本/` - 所有分镜脚本文件
- `02_角色设定/` - 角色设定卡（用于提示词参考）

## 输出

- 补充后的分镜脚本文件
- `03_分镜脚本/提示词补充报告.md`

---

## 执行步骤

### Step 1: 扫描所有分镜文件

读取 `03_分镜脚本/` 目录下所有 `.md` 文件，识别所有镜头。

```
扫描中...
- 第1集.md: 15 个镜头
- 第2集.md: 18 个镜头
- 第3集.md: 12 个镜头
总计: 45 个镜头
```

### Step 2: 逐镜头检查提示词完整性

对每个镜头执行以下检查：

| 检查项 | 通过标准 | 失败原因 |
|--------|----------|----------|
| 提示词存在 | 有"即梦AI视频提示词"或"视频提示词"字段 | 字段缺失 |
| 提示词非空 | 内容不为空、不是"待补充"、不是占位符 | 内容为空 |
| 提示词长度 | 字数 >= 50 字 | 内容过短 |
| 结构完整 | 包含场景+主体+动作+风格+运镜 | 结构缺失 |

### Step 4: 逐个补充提示词

对每个缺失/异常的镜头，执行以下流程：

**4.1 提取分镜信息**

从分镜脚本中提取该镜头的：
- 场景描述
- 时间设定
- 角色信息
- 动作描述
- 情绪氛围
- 运镜设计

**4.2 生成提示词**

根据提取的信息，生成完整的提示词：

```markdown
### S01_C05 提示词补充

**分镜信息提取：**
- 场景：办公室，现代化的开放式办公区
- 时间：下午
- 角色：林小雨，穿着白色衬衫，表情焦虑
- 动作：盯着电脑屏幕，手指快速敲击键盘，偶尔咬嘴唇
- 情绪：紧张、焦虑
- 运镜：中景，缓慢推进

**生成的提示词：**
```
下午时分的现代化办公室，阳光透过落地窗洒入，办公区内其他同事在忙碌。一位25岁的职场女性，穿着白色衬衫，扎着低马尾，眉头紧锁地盯着电脑屏幕，手指快速敲击键盘，偶尔停下来咬咬嘴唇，眼神中透露着焦虑和紧张。冷色调光线，中景构图，镜头从侧面缓慢推进。
```

**补充原因：** 原提示词为空
```

**4.3 写入文件**

将补充的提示词写入对应的分镜脚本文件。

## 提示词生成规范

补充的提示词必须符合以下规范：

### 结构要求
```
[场景描述] + [主体特征与动作] + [视觉风格] + [情绪或氛围] + [镜头语言]
```

### Seedance 2.0 格式（可选）
```
Subject: [场景核心主体]
Action: [主体动作]
Camera: [镜头尺寸+镜头运动+镜头角度]
Style: [视觉风格]，[灯光效果]
Constraints: [负面提示]，[视频时长]
```

### 质量标准
- 字数：50-150 字为宜
- 具体性：避免模糊描述，使用具体词汇
- 完整性：包含场景、人物、动作、风格、运镜五要素
- 连贯性：与前后镜头能自然衔接

## 常见问题处理

### 问题1：分镜描述本身不完整
```
情况：分镜只有简单的"林小雨在办公室"
处理：根据上下文和角色设定补充细节
```

### 问题2：角色信息缺失
```
情况：分镜中没有角色描述
处理：引用 `02_角色设定/` 中的角色卡信息
```

### 问题3：运镜信息缺失
```
情况：分镜没有运镜描述
处理：根据镜头情绪选择合适运镜：
- 紧张情绪 → 手持晃动 + 快速推近
- 温馨情绪 → 缓慢推进 + 柔和光线
- 震撼情绪 → 大范围拉远
```

### 问题4：无法确定场景细节
```
情况：场景描述过于模糊
处理：
1. 查看同集其他镜头的场景描述
2. 参考故事大纲中的场景设定
3. 使用通用场景描述
```
