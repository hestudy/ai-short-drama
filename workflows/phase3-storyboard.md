# 阶段三：分镜脚本工作流

## 目标

将故事转化为具体的分镜脚本，并为每个镜头生成即梦AI视频提示词。

## 输入

- `01_故事大纲.md`
- `02_角色设定/`

## 输出

- `03_分镜脚本/第X集.md` × N
- `04_视频提示词/第X集/` × N

---

## 执行步骤

### Step 1: 确定分镜策略

与用户确认：

```
分镜策略选择：

1. 【镜头密度】
   - 紧凑型：每分钟约8-10个镜头
   - 标准型：每分钟约5-7个镜头
   - 舒缓型：每分钟约3-5个镜头

2. 【叙事风格】
   - 对话驱动：多中近景，强调人物互动
   - 动作驱动：多动感镜头，强调运动感
   - 氛围驱动：多远景特写切换，强调情绪渲染

3. 【镜头时长偏好】
   - 快节奏：平均2-3秒/镜头
   - 中等：平均4-5秒/镜头
   - 慢节奏：平均6秒以上/镜头

请选择或说明您的偏好：
```

### Step 2: 逐集拆分分镜

对每一集执行：

#### 2.1 生成本集概要

```markdown
# 第X集：[标题]

## 集数信息
| 项目 | 内容 |
|------|------|
| 时长 | 秒 |
| 镜头数 | 个 |
| 核心事件 | |
| 情绪曲线 | |

## 剧情梗概
[200字以内的本集剧情概述]
```

#### 2.2 拆解镜头清单

```markdown
## 镜头清单

| 编号 | 场景 | 角色 | 动作 | 时长 | 情绪 |
|------|------|------|------|------|------|
| S01_C01 | | | | | |
| S01_C02 | | | | | |
| ... | | | | | |
```

#### 2.3 详细分镜描述

对每个镜头生成：

```markdown
### S01_C01 [简短描述]

**场景：** [详细场景描述]
**时间：** [时间]
**角色：** [出场角色及状态]
**动作：** [具体动作描述]
**情绪：** [角色情绪 + 画面氛围]
**运镜：** [景别 + 运镜方式]
**时长：** 秒

**即梦AI视频提示词：**
```
[完整提示词，格式：场景 + 主体动作 + 风格 + 情绪 + 运镜]
```
```

### Step 3: 生成提示词文件

为每一集创建：

```
04_视频提示词/第X集/
├── S0X_C01.txt
├── S0X_C02.txt
├── ...
└── 批量提示词.md
```

**单个提示词文件格式 (S0X_C0Y.txt)：**
```
# S01_C01 - [镜头简述]

## 角色参考
[如果需要上传角色参考图，说明使用哪个角色的哪张参考图]

## 场景参考
[如果需要上传场景参考图，说明描述]

## 视频提示词
[完整的即梦AI视频生成提示词]

## 备注
[特殊说明，如：需要与上一镜头保持动作连贯]
```

**批量提示词文件格式 (批量提示词.md)：**
```markdown
# 第X集视频提示词汇总

> 复制到即梦AI逐个生成视频片段

---

## 所需参考图清单

### 角色参考图
| 角色 | 用途 | 文件 |
|------|------|------|
| 角色A | 所有涉及该角色的镜头 | 角色A_正面标准照.png |

### 场景参考图（可选）
| 场景 | 用途 | 生成提示词 |
|------|------|------------|
| 场景A | C01, C05 | [提示词] |

---

## 视频提示词

### S01_C01
```
[提示词]
```
**时长建议：** 秒
**参考图：** 角色A正面标准照

### S01_C02
```
[提示词]
```
**时长建议：** 秒
**参考图：** 无

（依次列出所有镜头）

---

## 生成顺序建议

1. 先生成角色参考图
2. 按镜头顺序生成视频
3. 复杂镜头可生成多个版本选择
```

---

## 提示词质量检查

对每个提示词进行检查：

```
✅ 结构完整：包含场景、主体、动作、风格、情绪
✅ 描述具体：避免模糊词汇，使用具体描述
✅ 时长匹配：提示词内容量与镜头时长相符
✅ 连贯性：与前后镜头能自然衔接
✅ 可执行性：避免AI难以理解的动作
```

---

## 完成标准

- [ ] 所有集数都有完整的分镜脚本
- [ ] 每个镜头都有详细的分镜描述
- [ ] 每个镜头都有可用的视频提示词
- [ ] 提示词已按集汇总，方便复制使用
- [ ] 用户已确认分镜内容
